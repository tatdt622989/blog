![Claude Code 2026 六月更新四大新功能示意封面](cover.jpg)

最近一個月用 Claude Code 寫東西，明顯感覺它從「會幫你寫程式的工具」往「能放著自己長時間跑的助理」在靠攏。六月初一連串小版本（v2.1.166 到 v2.1.169，大約落在 6/6 到 6/8）更新了四個我覺得很實用、而且**馬上就能用上**的功能。這篇就用實際開發情境，帶你看懂每個功能在解決什麼痛點。

<!--more-->

## 先講結論：這次更新主軸是「穩定度」

這四個新功能乍看零散，但串起來其實是同一個方向：讓 Claude Code 在**長時間、多專案、會出意外**的真實工作裡更穩。換目錄不中斷、任務能分層外包、壞掉時能快速隔離問題、主力模型塞車時自動換備援——全都是為了讓你少被打斷。底下一個一個講。

## 一、/cd：對話到一半換工作目錄，不用重開

以前在一個 session 裡只能綁定一個工作目錄，要切到另一個專案就得重開、context 也跟著歸零。新的 `/cd` 指令可以**在對話中途直接把 session 移到另一個目錄**，而且關鍵是它**不會重建 prompt 快取**——新目錄的 **CLAUDE.md** 是用「附加一則訊息」的方式接進來，而不是整個把系統提示換掉。

對 monorepo 或同時開好幾個 package 的人來說超有感：在前端目錄問完問題，直接 `/cd` 到後端目錄繼續，前面累積的脈絡都還在，也不用再吃一次快取重建的延遲。

## 二、子代理可以再生子代理（但有上限）

這版開始，**子代理（sub-agent）能再往下生自己的子代理**，做分層委派。比方說主代理負責「重構整個模組」，它可以丟給一個子代理規劃，子代理再各自派下去處理單一檔案。

當然 Anthropic 有防呆：**背景**的子代理鏈最多疊**五層**，避免你一個不小心開出一棵失控、瘋狂併發的任務樹；**前景**的鏈則可以任意深度，因為它本身會自我節制。要把大任務拆給多個代理平行處理時，這個設計會讓分工乾淨很多。

## 三、--safe-mode：壞掉時，一鍵回到乾淨狀態

這個我私心最愛。用久了難免裝一堆 plugin、hook、MCP server、自訂指令，哪天 Claude Code 行為怪怪的，第一個問題永遠是「到底是它的 bug，還是我自己的設定搞的？」

現在用 `--safe-mode` 啟動（或設環境變數 `CLAUDE_CODE_SAFE_MODE`），它會**把所有自訂內容全部關掉**：**CLAUDE.md**、skills、plugins、hooks、MCP server、自訂指令與代理通通不載入。但登入、模型選擇、內建工具、權限這些**照常運作**。所以你能很快判斷：safe mode 下正常，就是你的某個自訂設定有問題，再回去一個一個開回來抓兇手，比盲猜快太多。

## 四、fallbackModel：主力模型塞車時自動換備援

碰過尖峰時段 Opus 排隊排到天荒地老的人都懂。新的 `fallbackModel` 設定可以**指定最多三個備援模型**，主力模型過載或不可用時，就照你排的順序自動往下換；而且 `--fallback-model` 現在**互動式 session 也吃得到**了。

如果你平常就在跟額度與尖峰塞車搏鬥，這個功能會很實用，搭配我之前整理的 [Claude 與 Codex 額度榨乾術](/2026/06/18/claude-codex-quota-guide/) 一起用，能少卡很多次。

## 小結：往「能自己長時間跑」再推一步

把這四個功能放在一起看，方向很清楚：**`/cd` 顧脈絡、巢狀子代理顧分工、`--safe-mode` 顧排錯、`fallbackModel` 顧可用性**。每一個都在補那種「放著讓它自己跑久一點」時會踩到的洞。
